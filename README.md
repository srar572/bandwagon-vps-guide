## 什么是 OpenVZ，什么是 KVM？一句话说清楚

KVM（Kernel-based Virtual Machine）是一种把物理服务器切割成独立虚拟机的技术——你的 VPS 有自己的内核，可以加载自定义模块，安装 Docker、BBR 这类需要内核支持的东西。

OpenVZ 的逻辑不一样。它是多个 VPS 共用同一个内核，隔离程度没 KVM 深，不支持自定义内核，所以很多操作做不了——你没有真正意义上的"根权限"。

这不是哪家厂商的问题，是技术架构决定的上限。

## 为什么 KVM 比 OpenVZ 好？差异集中在这几点

既然已经没 OpenVZ 可选，这部分主要是帮那些还不太清楚为什么的人补一下背景知识，方便你理解为什么 KVM 是更合理的选择。

**内核完全独立**：KVM 下每个 VPS 跑自己的 Linux 内核，可以加 BBR、可以跑 Docker、可以用 WireGuard。OpenVZ 共享宿主机内核，这些基本没法做。

**BBR 加速**：这个对国内用户特别有感知。BBR 是 Google 搞的 TCP 拥塞控制算法，开了之后晚高峰的速度和延迟会明显改善。KVM 直接支持，OpenVZ 做不到。搬瓦工的 KiwiVM 后台里甚至可以一键选择带 BBR 的系统模板，省去手动配置的麻烦。

**可以切换 CN2 机房**：这是很实际的一个点。搬瓦工的 KVM 套餐购买后可以在 KiwiVM 后台一键迁移到 CN2、CN2 GIA 等优化机房，而 OpenVZ 方案是无法迁入 CN2 机房的。

**系统支持更全**：KVM 下 Ubuntu、Debian、CentOS、Rocky Linux 全部正常跑，没有神秘限制。

## BBR 加速怎么开？一步到位

用搬瓦工 KVM 套餐有一个快捷路径——在 KiwiVM 后台重装系统时，选后缀带 `_bbr` 的模板（比如 `centos-7-x86_64-bbr`），直接预装 BBR，不用手动配置内核。

如果选的是 Debian 10/11/12 或 Ubuntu 20.04 以上，这些系统本身已经内置 BBR，重装完不用额外操作，开机就有。

OpenVZ 的时候完全没法这样玩。KVM 把这条路打开了。
